# Documentation instructions

Documentation records decisions, invariants, contracts, migration requirements, rejected alternatives and source evidence. Do not describe planned behavior as implemented. Link official Cloudflare documentation and pin referenced GitHub commits.

Before changing implementation guidance, read `docs/implementation-pathway.md`, `docs/reference-inventory.md`, all relevant ADRs and the applicable note under `docs/ports/`. A port note must distinguish copied, translated and behavior-derived work and must record licensing before code is imported.
